通过Gost建立Wss/Https代理

2022-08-21 202208 345 字
Gost CloudFlare

准备工作

  • 一台能连接Github的VPS.
  • 一个域名.

Notice

本文中的Gost为v2版本.

Server

下载Gost.

可以前往Gost releases获取最新版链接.

wget https://github.com/ginuerzh/gost/releases/download/v2.11.4/gost-linux-amd64-2.11.4.gz
#下载Gost源文件

gzip -d gost-linux-amd64-2.11.4.gz
#解压源文件

mv gost-linux-amd64-2.11.4 gost
#重命名为gost，方便后续使用命令

chmod +x gost
#使之可执行

设置Https代理

sudo nohup ./gost -L=https://username:password@:443?probe_resist=code:404\?cert=/path/to/your/cert/file\&key=/path/to/your/key/file

设置Websocket+TLS隧道

sudo nohup ./gost -L=mwss://username:password@:443?cert=/path/to/your/cert/file\&key=/path/to/your/key/file

补充说明

  • nohup：防止结束SSH后Gost程序终止.

  • probe_resist：防御探测，具体可见Gost Wiki.

  • 用户名/密码及证书地址记得替换.

Client

Https代理

可以直接在大部分软件中使用.

在clash中配置文件为：

- {name: "Name", server: example.com, port: 443, type: http, username: "username", password: "password", tls: true }

Websocket+TLS隧道

./gost -L socks://:1080 -F mwss://username:password@example.com:443
  • 推荐在WSL子系统中使用.
  • 规则可以在Windows中使用Clash来弥补.
    • 如果你是在WSL子系统中运行Gost Client，在CFW中使用的地址为localhost:1080.

CloudFlare CDN

  • CF Dashboard—SSL/TLS—Overview 『Full』.
  • Dashboard—DNS—Proxy status 『Proxied』.

Note: 套CloudFlare后可能会导致速度变慢，若非线路不佳可不开启Proxied.
