通过Gost建立Websocket Tunnel

2022-08-21 202208 182 字
Gost CloudFlare

准备工作

  1. 一台能连接Github的VPS.
  2. 一个域名.

Notice

本文中的Gost为v2版本.

Server

可以前往Gost releases获取最新版链接.

wget https://github.com/ginuerzh/gost/releases/download/v2.11.3/gost-linux-amd64-2.11.3.gz

gzip -d gost-linux-amd64-2.11.3.gz

mv gost-linux-amd64-2.11.3.gz gost

chmod +x ./gost

sudo nohup ./gost -L=mwss://username:password\@:443?probe_resist=code:404\&cert=....../fullchain.pem\&key=....../privkey.pem

补充说明

nohup：防止结束SSH后Gost程序终止.

probe_resist：防御探测，具体可见Gost Wiki.

用户名/密码及证书地址记得替换.

Client

./gost -L socks://:1080 -F mwss://username:password@example.com:443

CloudFlare CDN

在Dashboard中设置好DNS Record并将Proxy status设为Proxied即可.

Note: 套CloudFlare后可能会导致速度变慢，若非线路不佳可不开启Proxied.
